## 命名规范

- **类名 (Classes):** 使用 `PascalCase`（大驼峰）。如：`ChassisController`, `DjiMotor`。
    
- **函数名 (Functions):** 使用 `snake_case`（下划线）。如：`set_target_velocity()`, `update_feedback()`。
    
- **变量名 (Variables):**  局部变量：`snake_case`。
    
- 成员变量：`m_` 前缀 + `snake_case`。如：`m_target_angle`。
    
- **常量/宏 (Constants):** 全大写 + 下划线。如：`MAX_WHEEL_SPEED`。
    
- **命名空间 (Namespace):** 全小写。如：`easy_robot::gimbal`。

## 目录规范

核心目录规范展示

```text
EasyRobot/
├── CMakeLists.txt          # 全局构建脚本 (CMake 构建入口)
├── README.md               # 项目简介与快速上手指南
├── config/                 # ⚙️ 配置文件区 (硬件绑定与参数)
├── bsp/                    # 🔌 板级支持包 (Board Support Package)
│   ├── can/                # CAN 底层收发封装
│   ├── uart/               # 串口/DMA 底层封装
│   └── spi/                # SPI 底层封装
├── components/             # 🛠️ 硬件驱动层 (传感器与执行器)
│   ├── motor/              # 电机驱动 (M3508, GM6020, 达妙等)
│   ├── imu/                # 惯导模块 (BMI088, MPU6500)
│   └── remote/             # 遥控器与图传链路 (DR16)
├── app/                    # 🚀 应用层 (机器人具体业务逻辑)
│   ├── chassis/            # 底盘控制任务
│   ├── gimbal/             # 云台控制任务
│   ├── shoot/              # 发射机构任务
│   └── main.cpp            # 程序主入口
```

不要全部文件都放在 Core/Src 目录下，如果是硬件驱动就放在 components ，CAN、USART 的封装放在 bsp，任务放到 app
